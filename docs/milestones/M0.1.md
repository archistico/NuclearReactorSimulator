# M0.1 — Engineering Foundation & Architectural Baseline

## Goal

Create a buildable, testable engineering baseline before introducing any reactor physics.

## Scope

- .NET 10 solution and project structure;
- Avalonia presentation shell;
- centralized package versions and compiler rules;
- nullable reference types and warnings-as-errors;
- clean dependency graph;
- composition root owned by App;
- architecture enforcement tests;
- initial ADRs and M0–M9 roadmap.

## Explicitly out of scope

- simulation clock;
- commands and snapshots;
- reactor physics;
- thermodynamics;
- fluid networks;
- persistence;
- scenarios;
- production control-room UI.

These belong to later milestones and must not be pulled prematurely into M0.1.

## Acceptance checklist

Run from the repository root:

```powershell
dotnet restore
dotnet build --no-restore
dotnet test --no-build
```

Expected validation:

1. restore succeeds;
2. build succeeds with zero warnings and zero errors;
3. all tests pass;
4. Avalonia application starts and shows the M0.1 foundation screen;
5. no production project except `NuclearReactorSimulator.App` references Avalonia;
6. architecture tests confirm the approved project-reference graph.
