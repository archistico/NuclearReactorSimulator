# M6.1 — Control-Room Application Shell

Status: **VALIDATED**

## Scope

- scalable Avalonia control-room shell with stable workspace navigation;
- snapshot-driven Application presentation contracts;
- command-dispatch boundary with no UI physics;
- presentation performance budgets independent from simulation timestep;
- architecture hardening that removes the Avalonia project's direct Simulation reference.

## Acceptance criteria

1. The shell exposes stable Overview, Reactor, Primary, Turbine/Secondary, Electrical and Alarms/Events workspaces.
2. Avalonia view models update from `ControlRoomSnapshot` rather than authoritative full-plant state.
3. Avalonia source has no direct `NuclearReactorSimulator.Simulation` namespace dependency.
4. Operator shell commands are dispatched through `IControlRoomCommandDispatcher`; views/view models do not execute plant physics.
5. Snapshot projection uses M5.7 measured/protection/alarm boundaries and exposes no `FullPlantSnapshot` or conserved physical state.
6. The desktop layout remains usable at the defined minimum window size and scales to larger displays.
7. UI performance budgets are explicit presentation targets only and cannot modify simulation timestep/results.
8. Detailed instruments, mnemonics, trends and annunciator widgets remain deferred to M6.2–M6.6.
