# M6.3 — Reactor/Core Panel

Status: **VALIDATED**

## Scope

- first domain-specific production control-room workspace;
- measured reactor thermal-power indication with M5.1 validity/quality semantics;
- explicit model diagnostics for reactor period, reactivity, rod state and aggregated core zones;
- coarse core-zone map with power, fuel/coolant temperature and void diagnostics;
- reactor SCRAM and rod-withdrawal-interlock presentation;
- canonical rod target selection plus insert/hold/withdraw command intents;
- SCRAM and protection-reset command intents through Application boundaries;
- explicit unavailable xenon presentation because M2.8 xenon state is not yet in the M5.7 operational envelope.

## Acceptance criteria

1. The Reactor workspace is a real domain panel composed from M6.2 reusable components.
2. Avalonia references only Application presentation types and contains no reactor physics/protection calculations.
3. Reactor thermal power is sourced from the measured frame when a valid semantic source channel exists; invalid/missing measurement is shown unavailable.
4. Period/reactivity/rod/core-zone values are explicitly identified as model diagnostics and remain presentation-only.
5. The coarse core-zone view shows canonical zone identity plus power, temperature and void diagnostics without inventing spatial physics.
6. Reactor SCRAM and rod-withdrawal inhibit reflect M5.5 protection presentation state.
7. Rod insert/hold/withdraw, SCRAM and protection-reset controls emit typed Application commands only.
8. Runtime-dependent controls fail closed while the shell runtime coordinator is not connected.
9. Missing xenon operational state is represented as unavailable rather than synthesized.
10. M6.2 is recorded as validated and M6.4 Primary-Circuit Mnemonics becomes the next milestone.


## Validation result

Locally validated by the user: compilation and all tests passed. M6.3 is the baseline for M6.4.
