# M10.9.3 — Interactive Full-Plant Mimic

**Status:** VALIDATED  
**Prerequisite baseline:** M10.9.2 Hotfix 2 — Advanced Instrument & Gauge System — VALIDATED

## Goal

Make the whole plant understandable as one connected process/energy system before the operator drills into subsystem workspaces.

M10.9.3 adds a persistent engineering mimic to the PLANT overview with selectable equipment, explicit input/output semantics, directional medium-aware connections, key operating data and direct subsystem drill-down.

## Architectural boundary

The mimic is split deliberately:

- `NuclearReactorSimulator.Application` composes immutable mimic presentation contracts from the already validated `ControlRoomSnapshot` boundary;
- Avalonia renders those contracts and publishes only the selected presentation element id;
- Avalonia does not infer topology, flow, phase, pressure, temperature, alarm state or process causality;
- no new physics, hydraulic/electrical network, controller, alarm or protection ownership is introduced.

The high-level mimic is a presentation composition of existing canonical subsystems, not a second plant model.

## Whole-plant equipment map

The overview publishes eight macro-equipment groups:

1. reactor core;
2. main circulation;
3. steam drums;
4. steam turbine;
5. generator;
6. external grid;
7. condenser/hotwell;
8. condensate/feedwater.

This level is intentionally compact. Detailed pump/valve/stage/channel instrumentation remains in the existing subsystem workspaces and is expanded graphically in M10.9.4.

## Process and energy connections

Nine explicit directed connections cover the complete high-level cycle:

```text
MAIN CIRCULATION → REACTOR → STEAM DRUMS → TURBINE → GENERATOR → GRID
        ↑                         │             │
        └──── LIQUID RETURN ─────┘             ↓
                                      CONDENSER → FEEDWATER → STEAM DRUMS
```

Connection semantics distinguish:

- primary coolant;
- steam;
- condensate;
- feedwater;
- mechanical shaft power;
- electrical export.

Each connection carries the best already-published presentation evidence available for that path, such as flow, pressure, temperature, speed, power, frequency or voltage. Missing evidence remains unavailable; the mimic does not fabricate it.

## Equipment presentation

Each equipment card exposes:

- recognizable equipment glyph;
- canonical equipment/system identity;
- state/status;
- one or two key operating values;
- explicit `IN` description;
- explicit `OUT` description.

Selection emphasizes the equipment and its directly connected paths.

## Drill-down

Selecting equipment updates the contextual summary and `OPEN SUBSYSTEM` routes to the existing canonical workspace:

- reactor → REACTOR;
- circulation/drums → PRIMARY;
- turbine/condenser/feedwater → TURBINE;
- generator/grid → GRID.

Drill-down changes presentation navigation only and has no plant-state side effect.

## Visual language

- primary coolant: cyan;
- steam: pale high-contrast vapor line;
- condensate: blue;
- feedwater: green-cyan;
- shaft/mechanical transfer: amber;
- electrical export: violet;
- warning/trip state remains semantically distinct from process-medium color.

Red remains reserved for alarm/trip/protection significance.

## Replay/fingerprint

M10.9.3 does not modify authoritative simulation snapshots or replay inputs. The mimic is derived from the existing control-room presentation snapshot and therefore does not expand fingerprint-v1 state.

## Automated coverage

Tests verify:

- eight whole-plant equipment groups;
- nine directed connections;
- all six medium/energy classes;
- explicit input/output text;
- route geometry presence;
- shell-only unavailable behavior;
- two-way XAML selection binding;
- subsystem drill-down command binding;
- visible medium legend and selected-equipment port/context bindings.

## Manual validation

After clean build/full tests, verify:

1. PLANT opens with the whole-plant mimic as the first major situation-awareness surface.
2. Equipment is visually distinguishable and names remain readable at minimum supported window size.
3. Arrows make flow/energy direction unambiguous.
4. Medium colors are distinguishable without using red as a normal process color.
5. Each selected equipment item clearly shows `IN`, `OUT`, key data and context.
6. Selecting an element visually emphasizes its connected paths.
7. Tab focus can traverse equipment cards; focusing a card selects it without issuing a plant command.
8. `OPEN SUBSYSTEM` navigates to the correct existing workspace without changing plant state.
9. Values update with runtime snapshots and unavailable data remains visibly unavailable.
10. Existing M10.9.2 gauges, M10.8 computer, keyboard operation and controls remain usable.

## Out of scope

M10.9.3 does **not** add:

- detailed subsystem engineering schematics, valve/pump-stage detail and richer temperature/phase visual encoding — M10.9.4;
- command-consequence chains — M10.9.5;
- external demand/scoring/challenge semantics — M10.9.6/M10.9.7;
- predictive UI-side physics;
- new plant topology or equipment ownership.
