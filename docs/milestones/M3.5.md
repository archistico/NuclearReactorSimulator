# M3.5 — Main Circulation System

## Objective

Compose the first semantic RBMK-like recirculation loops from the canonical plant topology, validated pump/pipe physics and M3.4 equivalent fuel-channel groups.

## Implemented

- `MainCirculationBranchDefinition`;
- `MainCirculationLoopDefinition`;
- `MainCirculationSystemDefinition`;
- eager validation of headers, pump endpoints, group assignments and return-path closure;
- one-loop ownership for every fuel-channel group;
- committed-state `MainCirculationSystemSolver`;
- pump, branch, loop and system immutable diagnostic snapshots;
- pump/channel/return flow and continuity diagnostics;
- local outlet phase/quality/void diagnostics;
- order-independent semantic definitions;
- integration tests confirming that `PlantNetworkOrchestrator` remains the only state-integration boundary.

## Explicitly deferred

- steam drums and separation: M3.6;
- feedwater/steam boundaries: M3.7;
- pump inertia/coastdown and electrical motors: M5/M8 or later fidelity work;
- detailed two-phase pressure-drop/cavitation models;
- turbine island.

## Validation gate

Local `dotnet build` and complete `dotnet test` must pass before M3.5 becomes a validated baseline.
