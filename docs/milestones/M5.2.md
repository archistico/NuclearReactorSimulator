# M5.2 — Controller & Actuator Primitives

## Goal

Provide reusable deterministic P/PI/PID and actuator-command primitives on top of the validated M5.1 measured-signal boundary, without yet adding plant-specific automatic loops.

## Delivered

- P/PI/PID definitions and fixed-step state evolution;
- manual/automatic modes;
- bounded output and explicit saturation diagnostics;
- conditional-integration anti-windup;
- bumpless manual-to-automatic transfer;
- deterministic invalid/unavailable-measurement hold behavior;
- typed valve, pump and control-rod actuator command seams;
- command-side actuator memory separate from physical plant state;
- controller/actuator composition from `MeasuredSignalFrame` only;
- domain and simulation verification;
- ADR 0040 and controller/actuator architecture documentation.

## Intentionally deferred

M5.3 owns reactor/primary-system loop wiring and M5.4 owns turbine/steam/feedwater loop wiring. Trips/SCRAM/interlocks remain M5.5.

## Validation

Run:

```powershell
dotnet restore
dotnet build --no-restore
dotnet test --no-build
```

M5.2 was locally validated after successful compilation and complete automated test execution. It is the validated baseline for M5.3.
