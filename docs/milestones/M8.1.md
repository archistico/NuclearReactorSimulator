# M8.1 — Deterministic Fault-Injection Framework

**Status:** BASELINE CANDIDATE

## Scope

- explicit immutable fault declarations attached to versioned `ScenarioDefinition` documents;
- stable fault ID, fault-type ID, target ID and deterministic string parameters without hidden randomness;
- activation and optional deactivation by exact logical-step boundary or named plant-condition evaluator;
- fail-closed exact-ID registries for plant-condition evaluators and runtime-bound fault applicators;
- deterministic `Pending → Active → Cleared` lifecycle with monotonic transition sequence and logical-step stamps;
- fault lifecycle projected into every `ControlRoomSnapshot` produced by a fault-enabled scenario session;
- scenario JSON schema v2 with deterministic v0/v1 migration that preserves exact initial-condition identity and invents no faults;
- replay reconstruction from the same versioned scenario plus operator command trace, without a parallel wall-clock/fault trace.

## Acceptance

- no M8.1 scheduling path uses wall clock, timers, random generators or hidden probabilistic activation;
- logical-step triggers apply at the exact committed boundary before that physical step executes;
- plant-condition triggers evaluate only the committed UI-safe `ControlRoomSnapshot`; they do not traverse authoritative true state;
- missing fault applicators or named condition evaluators fail session loading closed;
- concrete subsystem effects are delegated to registered typed applicators and M8.1 itself owns no pump/valve/sensor/controller physics;
- replaying the same exact initial condition, fault schedule and operator command trace reproduces identical fault lifecycle state;
- pre-M8 scenario documents migrate with an empty fault list rather than receiving inferred behavior;
- complete build and automated suite must pass locally before M8.1 validation.

## Out of scope

Concrete hydraulic component faults remain M8.2. Sensor/control-loop/interlock diagnostic faults remain M8.3. Turbine/generator/feedwater/condenser transients remain M8.4. Leak/LOCA-class and electrical-loss/SBO-class scenario physics remain M8.5/M8.6. M8.1 does not force scripted plant outcomes, create a second physical state owner or implement arbitrary full-state checkpoint/seek functionality.
