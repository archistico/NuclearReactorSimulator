# M3.4 — Fuel-Channel Group Model

## Goal

Turn aggregated core zones into operational equivalent fuel-channel groups while preserving global point kinetics and the staged plant-network orchestration boundary.

## Delivered

- `FuelChannelGroupDefinition` and canonical `FuelChannelGroupSetDefinition`.
- Represented channel counts and per-zone group power fractions.
- Validation that every core zone has groups whose power shares sum to 1.0.
- Validation of canonical hydraulic pipe, inlet/outlet nodes and zone thermal domains.
- Explicit fuel / structure / coolant heat-deposition fractions.
- `FuelChannelGroupSolver` with exact deterministic fission-power partitioning.
- Optional global decay-heat routing through the same spatial/group shape while retaining separate diagnostics.
- Per-group hydraulic-flow diagnostics from the committed state using the validated passive pipe solver.
- Per-channel power and mass-flow diagnostics.
- `PlantNetworkSourceTerms` seam for staged external physical source terms.
- M3.2 orchestrator support for supplemental balances and explicit supplemental external-power audit.
- Tests for canonical ordering, invalid topology, exact power closure, heat routing, hydraulic diagnostics and composed plant-network energy audit.

## Not included

- Main circulation headers/pumps integration: M3.5.
- Steam drums and separation: M3.6.
- Individual fuel channels or spatial neutron diffusion: later fidelity work.

## Exit criteria

Validated by successful local `dotnet build` and complete `dotnet test`; M3.4 is the stable baseline carried into M3.5.
