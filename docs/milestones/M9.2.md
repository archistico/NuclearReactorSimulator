# M9.2 — Post-Incident Analysis

**Status:** VALIDATED  
**Prerequisite baseline:** M9.1 — Recorder, Checkpoints & Full Replay — VALIDATED

## Goal

Provide deterministic post-incident debrief analysis over immutable M9.1 recordings without introducing new physical, control, protection or fault ownership.

## Delivered

- `ScenarioPostIncidentAnalyzer` over complete `ScenarioRecording` data.
- deterministic incident-anchor selection, with optional exact recorder-event sequence override;
- logical-step pre/post analysis windows only; no wall-clock timing;
- evidence timeline preserving recorder sequence, logical step, relative step and before/anchor/after relation;
- synchronized fixed-step trend samples from values already present in `ControlRoomSnapshot`, plus compact state summaries at window start, anchor and window end;
- response metrics for first alarm, first protection activation, first operator action and first fault clear;
- peak invalid-signal, alarm and active-fault counts inside the selected window;
- nearest preceding replay-backed checkpoint reference for verified reconstruction;
- versioned `PostIncidentAnalysisReport` schema v1 and JSON persistence adapter;
- fail-closed unknown schema handling.

## Interpretation rule

M9.2 reports **observed facts and temporal relationships**. It does not infer physical causality from event ordering alone. A report may state that one event preceded another by N logical steps; it must not silently claim that the first caused the second unless that relationship is explicitly represented by the source model.

## Non-goals

- no new plant solver or state owner;
- no accident-specific physics;
- no automatic root-cause assertion;
- no wall-clock timestamps or UI-refresh-dependent metrics;
- no mutation of recordings, checkpoints or replay results;
- no replacement for M9.1 replay verification;
- no hidden access to private conservation/audit objects that are not present in the M9.1 recording contract.

## Validation gate

M9.2 is **VALIDATED**: the user explicitly confirmed successful local compilation and the complete automated test suite. It was the official functional baseline carried into M9.3 preparation; M9.3 and then M9.4 were subsequently validated, with M9.4 now the current baseline.
