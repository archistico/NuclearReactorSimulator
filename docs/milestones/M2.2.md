# M2.2 — Control Rods

## Goal

Introduce deterministic control-rod and group mechanics, persistent motion commands, mechanical limits and explicit position-to-reactivity mapping while preserving the M2.1 boundary that reactivity is only an input to future neutron kinetics.

## Delivered

- normalized `ControlRodPosition` (`0 = inserted`, `1 = withdrawn`);
- normalized `ControlRodTravelRate` and full-travel-time conversion;
- persistent `Hold`, `Insert` and `Withdraw` motion state;
- immutable rod, group and complete-system definitions;
- canonical immutable system state;
- individual-rod and group command targeting;
- deterministic same-step command override ordering;
- fixed-step rod movement with endpoint clamping and automatic hold;
- linear and smooth-step integral worth curves;
- canonical one-per-rod `ControlRods` reactivity contributions;
- composition through the validated M2.1 `ReactivityModel`;
- fixed-step/pulse-segmentation runtime determinism tests;
- ADR 0016 and control-rod documentation.

## Explicitly deferred

- neutron point kinetics and delayed-neutron groups;
- reactor period;
- physical axial rod geometry;
- RBMK graphite-displacer transient detail;
- automatic rod regulation and SCRAM systems;
- direct thermal-power response.

## Acceptance checklist

- solution restores successfully;
- solution builds with warnings treated as errors;
- all tests pass;
- position boundaries and travel-rate validation are enforced;
- insert/withdraw motion is deterministic and respects mechanical limits;
- group commands affect only group members;
- later same-step commands deterministically override earlier overlapping commands;
- worth curves preserve configured endpoint reactivities;
- one named reactivity contribution is produced per rod;
- external pulse segmentation does not alter final rod position or rod-reactivity snapshot;
- no direct rod-position-to-power or rod-position-to-neutron shortcut exists.
