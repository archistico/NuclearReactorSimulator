# M7.3 — First Criticality & Low-Power Operation

## Objective

Provide the first controlled operator progression from a prepared pre-criticality/source-range state through approach to criticality and educational low-power stabilization, while preserving canonical M2/M3/M5 ownership and keeping steam/turbine/grid startup outside the milestone.

## Delivered

1. Exact built-in initial-condition identity `pre-criticality-source-range` version `1`.
2. `FirstCriticalityInitialConditionFactory` reusing the validated M7.2 construction path instead of duplicating the canonical M1–M5 runtime object graph.
3. Prepared pre-criticality handoff with main circulation established, rods inserted, steam admission isolated, turbine stopped, generator disconnected and a deterministic non-zero source-range point-kinetics seed.
4. Explicit modeling note that the source-range seed is initial-condition data for the existing homogeneous M2 kinetics, not a hidden external-neutron-source solver.
5. M7.3 scenario permissions enabling rod INSERT/HOLD/WITHDRAW and safety actions while continuing to reject breaker close, turbine-speed and generator-load commands.
6. `FirstCriticalityChecklistEvaluator` observing only immutable `ControlRoomSnapshot` data for signal health, protection, circulation, isolation, rod permissive, source-range power, approach reactivity, criticality, low-power band and reactor-period stabilization.
7. Ordered declarative first-criticality guidance with controlled withdrawal, HOLD near criticality, low-power stabilization and an explicit M7.4 handoff; no automatic command dispatch or state forcing.
8. Explicit xenon-boundary objective: quantitative xenon remains `Unavailable` because M2.8 state is not yet promoted into the M5.7 operational envelope; M7.3 does not fabricate or privately integrate it.
9. Desktop composition loading the exact M7.3 pre-criticality/source-range session paused through the M7.1 registry/session boundary.
10. Application tests for exact-version reconstruction, initial pre-criticality observables, scenario action gating, rod-command progression, guidance ordering and checklist state.
11. ADR 0055 and `docs/FIRST_CRITICALITY_LOW_POWER.md`.

## Non-goals

- no external startup neutron-source physics model;
- no automatic rod-withdrawal procedure or forced criticality;
- no quantitative xenon integration inside scenario/UI layers;
- no primary heat-up or steam-pressure raising procedure;
- no turbine warm-up/acceleration;
- no generator synchronization, breaker closure or load increase;
- no fault injection or hidden randomness;
- no duplicate physical state owner.

## Validation status

**BASELINE CANDIDATE.** M7.2 (including hotfix 1) was explicitly validated by the user with successful local compilation and complete tests. M7.3 requires local `dotnet build` and the complete automated test suite before it becomes the new validated baseline.
