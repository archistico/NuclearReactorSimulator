# M1.1.1 — Nullable Architecture Test Hotfix

## Status

Validated — local build and complete test suite passed.

## Reason

The M1.1 local build exposed `CS8619` in `ArchitectureRulesTests.ReadProjectReferences`: `Path.GetFileNameWithoutExtension` is annotated by the BCL as returning `string?`, so the LINQ projection was inferred as `string?[]` while the method contract returns `string[]`. Because the solution treats warnings as errors, compilation correctly stopped.

## Fix

Project-reference filename extraction now handles the nullable BCL contract explicitly and fails fast if a project name cannot be derived.

No production code, physical quantity semantics, public APIs or simulation behavior changed.

## Acceptance

```powershell
dotnet restore
dotnet build --no-restore
dotnet test --no-build
```
