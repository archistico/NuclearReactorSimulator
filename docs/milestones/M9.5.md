# M9.5 — Historical-Inspired Scenario Framework

## Goal

Add an explicit provenance and model-fidelity gate before introducing historical-inspired initial conditions or procedures.

The milestone must separate documented facts, educational approximations and simulator-specific assumptions while preserving the canonical scenario/runtime ownership established by M7–M9.4.

## Status

**VALIDATED.**

The user explicitly confirmed local compilation and the complete automated test suite passed. M9.5 is the official validated baseline for M9.6.

## Delivered contracts

Application:

- optional `ScenarioDefinition.HistoricalContext`;
- `HistoricalSourceReference`;

- `HistoricalScenarioClaimDefinition`;

- `HistoricalScenarioFidelityReviewer`;

Infrastructure:

- scenario JSON schema v3 with optional historical context persistence;
- v0/v1/v2 migration without invented historical metadata.

## Claim classes

`DocumentedFact`

- must reference at least one declared source;
- represents only the statement the scenario author explicitly classifies as source-backed.

`EducationalApproximation`

- requires an explicit rationale;
- documents deliberate reduction/simplification caused by current simulator fidelity or training goals.

`SimulatorSpecificAssumption`

- requires an explicit rationale;
- documents deterministic values/choices introduced by the simulator where the source record/model does not uniquely define them.

No category is allowed to silently masquerade as another.

## Fidelity review

A historical-inspired scenario declares stable required model-capability IDs.

Before session runtime creation, `ScenarioSessionFactory` invokes `HistoricalScenarioFidelityReviewer` against the active validated capability set. Missing requirements fail closed with `HistoricalScenarioFidelityException`.

The current capability catalog describes validated capabilities through M9.4. It is a fidelity declaration boundary, not a second physics/control owner and not a runtime feature-toggle system.

## Historical interpretation rule

M9.5 establishes a framework for **historical inspiration**, not automatic historical reconstruction.

Every context requires:

- a fidelity statement;
- at least one source-backed documented fact;
- explicit claim classification;
- at least one explicit required model capability;
- at least one deliberate non-claim.

A passed fidelity review means only that the declared simulator capabilities exist. It does not validate historical truth or quantitative agreement.

## Scenario ownership rule

Historical metadata is observational/descriptive.

It may not:

- write physical state;
- create scenario-specific reactor/thermal/electrical physics;
- bypass canonical controller/protection ownership;
- script target trajectories or outcomes;
- infer causality from chronology;
- substitute source claims for model behavior.

Historical-inspired scenarios continue to use exact initial-condition versions, typed commands, deterministic M8 fault seams and canonical M2–M5 dynamics.

## Compatibility

Existing v0/v1/v2 scenario documents deserialize with no historical context.

Existing in-memory scenario definitions remain valid because `HistoricalContext` is optional.

M7/M8/M9.1 replay semantics are not silently changed: old scenarios are not automatically reclassified as historical-inspired and no historical data is invented during migration.

## Test coverage

Regression coverage includes:

- documented facts require source references;
- historical contexts require at least one documented fact, one explicit model capability and one deliberate non-claim;
- claim references to undeclared sources are rejected;
- fact/approximation/assumption categories remain explicit;
- missing required capabilities produce a deterministic failed review;
- complete explicit capabilities produce approval;
- ordinary scenarios without historical context cannot be reviewed as historical-inspired;
- session loading blocks historical content before runtime creation when fidelity requirements are unmet;
- scenario schema v3 round-trips historical context;
- schema v2 migration preserves scenario identity and yields no invented historical context.

## Deliberate non-goals

M9.5 does not introduce:

- a built-in named historical accident reconstruction;
- source retrieval/network access;
- source authority ranking;
- quantitative historical calibration;
- scenario-owned physical outcomes;
- licensing-grade historical fidelity claims.

Those boundaries remain explicit prerequisites for later content/calibration work.
