# M6.4 — Primary-Circuit Mnemonics

Status: **VALIDATED**

## Scope

- topology-aware primary-circuit mnemonic over canonical M3 loop/channel/drum definitions;
- measured loop total flow and header-pressure-rise instruments;
- measured steam-drum pressure and level instruments;
- explicitly labelled header/MCP/branch/drum/boundary model diagnostics;
- canonical primary-connected valve mechanical-state presentation;
- typed MCP start/run and stop operator command intents through Application boundaries;
- presentation-only flow-direction semantics.

## Acceptance criteria

1. The Primary Circuit workspace is a real domain mnemonic using M6.2 reusable controls.
2. Avalonia consumes only Application presentation records and contains no hydraulic/thermodynamic calculations.
3. Loop flow/header ΔP and drum pressure/level come from M5.1 measured channels and fail unavailable when channels are invalid/missing.
4. Absolute pressures, MCP/branch details, drum temperature/flows and other non-instrumented data are explicitly labelled model diagnostics.
5. Loop, pump, channel-group and drum identities follow canonical M3 topology.
6. Valve presentation includes only canonical valves connected to primary-circuit nodes and permits an empty set.
7. MCP controls emit typed Application pump intents only; no UI code mutates `PumpState` or executes M3/M5 solvers.
8. Runtime-dependent MCP controls fail closed while the runtime coordinator is unavailable.
9. M6.3 is recorded as validated and M6.5 Turbine, Generator & Electrical Panels becomes the next milestone.


## Validation result

Validated locally by the user: compilation and complete test suite passed. M6.4 is the baseline for M6.5.
