# M7.1 — Versioned Initial Conditions & Scenario Framework

## Objective

Open the M7 operations/training phase by establishing deterministic, versioned ownership for initial-condition reconstruction, scenario metadata, session load/start and replay boundaries without inventing procedure logic or duplicating physical state ownership.

## Delivered

1. Exact `InitialConditionReference` identity using immutable `(InitialConditionId, Version)` pairs.
2. `InitialConditionDescriptor` metadata plus `IVersionedInitialConditionFactory` construction seam.
3. `VersionedInitialConditionRegistry` with exact-version resolution, duplicate rejection and no implicit latest-version fallback.
4. Immutable `ScenarioDefinition` with metadata, descriptive objectives and explicit allowed operator actions.
5. `ScenarioCommandDispatcher` fail-closed operator-action gating while leaving run/pause/single-step under runtime-host ownership.
6. `ScenarioSessionFactory` canonical load/start boundary creating a fresh runtime and paused coordinator from one exact initial-condition version.
7. `ScenarioReplayRunner` over the validated M0 `SimulationCommandTrace<ControlRoomCommand>` with logical-step-only ordering.
8. Infrastructure JSON scenario schema version `1` plus deterministic legacy `v0 → v1` migration preserving the exact initial-condition ID/version.
9. Tests covering exact-version resolution, paused fresh session loading, disallowed-action rejection, deterministic replay, canonical schema round-trip, legacy migration and unknown-future-version rejection.
10. ADR 0053 and `docs/INITIAL_CONDITIONS_SCENARIO_FRAMEWORK.md`.

## Non-goals

- no concrete cold-shutdown/pre-start recipe yet — M7.2 owns the first operational initial condition;
- no procedure scripting or guided startup sequence;
- no objective scoring/evaluation engine;
- no fault scheduler;
- no arbitrary full-state checkpoint/save/seek format;
- no physics or state reconstruction in Avalonia.
