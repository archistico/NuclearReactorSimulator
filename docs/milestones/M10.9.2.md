# M10.9.2 — Advanced Instrument & Gauge System

**Status:** VALIDATED — HOTFIX 2  
**Prerequisite baseline:** M10.9.1 — HMI Information Architecture & Visual Language — VALIDATED

## Hotfix 2 — measured-instrument XAML label contract restoration

After Hotfix 1 compiled, the complete test suite exposed one App-layer regression: `MainWindowXamlContractTests.MeasuredAndModelLabels_RemainExplicitOnKeyInstrumentation` requires canonical static XAML provenance labels on key instrumentation. The new gauges had shortened `ROTOR SPEED · MEASURED` to `ROTOR SPEED` and `ELECTRICAL OUTPUT · MEASURED` to `ELECTRICAL OUTPUT`, relying on runtime provenance badges for `MEASURED`. The first failed assertion exposed rotor speed; static review of the remaining assertions identified electrical output before another test cycle.

Hotfix 2 restores both canonical XAML labels while retaining the gauge provenance badges and all M10.9.2 instrument semantics. No projection, threshold, trend, replay, physics, protection, command or control behavior changes.

## Hotfix 1 — nullable setpoint compile correction

The first M10.9.2 candidate exposed two C# `CS0173` compile errors in `ControlRoomSnapshotProjector`: local variables for steam-drum level and steam-pressure controller setpoints used `var` with conditional expressions mixing `null` and `double`.

Hotfix 1 changes only the local variable declarations to explicit `double?`. The numeric transformations, `WithinScale(...)` filtering, canonical setpoint ownership and all gauge semantics are unchanged.

## Goal

Turn important scalar process values into readable engineering instruments that answer more than “what is the number?” without moving threshold, protection or physics ownership into Avalonia.

M10.9.2 introduces reusable banded linear/circular instruments, deterministic logical-step trend indication, explicit provenance/quality/off-scale semantics and canonical scale metadata projection for selected operational quantities.

## Core rule

The HMI keeps these concepts independent:

```text
display / instrument scale
    ≠ operating interpretation bands
    ≠ target window
    ≠ controller setpoint
    ≠ alarm thresholds
    ≠ protection / trip limits
```

A value being **on scale never means it is safe or acceptable**.

## New presentation semantics

`ControlRoomValueSnapshot` gains presentation-only metadata for:

- scale metadata;
- `MEASURED` / `MODEL` / annunciator provenance;
- signal/display quality;
- explicit below-scale / above-scale detection.

These additions are excluded from serialized fingerprint-v1 state because they are derivable presentation semantics, not authoritative plant state.

`ControlRoomInstrumentTrendSnapshot` derives a display trend only from two published logical-step samples. It never reads wall clock time and returns unavailable across same-step publication or runtime/session reset.

## Gauge controls

### Linear gauge

Used when relative position along a bounded engineering scale is the clearest representation. It supports:

- scale minimum/maximum;
- canonical operating/warning/alarm bands when published;
- target window;
- setpoint marker;
- protection-limit markers;
- current-value marker;
- numeric contextual legend for published bands/targets/setpoints/protection thresholds;
- explicit off-scale arrow/status;
- provenance and quality badges;
- optional deterministic logical-step trend.

### Circular gauge

Used selectively where dial position materially improves operator awareness. M10.9.2 uses it for:

- reactor thermal power;
- turbine rotor speed.

Circular gauges are not used decoratively for every scalar value.

## Canonical metadata projection

M10.9.2 publishes gauge semantics from existing authoritative definitions/contracts:

- **reactor thermal power:** measured value, reference-derived display scale and existing reactor-power controller setpoint;
- **average rod withdrawal:** physical 0–100% position scale;
- **steam-drum pressure:** measured value plus matching M5 alarm thresholds, M5 protection thresholds and existing steam-pressure controller setpoint;
- **steam-drum level:** physical 0–100% level scale plus existing drum-level controller setpoint;
- **turbine rotor speed:** measured value plus canonical rated/overspeed-domain scale, existing speed setpoint and canonical overspeed protection marker;
- **generator frequency / terminal voltage / phase difference:** existing synchronization tolerances become explicit target windows around grid references;
- **generator electrical output / gross output:** canonical configured generator-capacity display scales.

No Avalonia control owns or invents these thresholds.

Where no defensible canonical scale exists yet, the value remains a numeric indicator. In particular, total turbine shaft power is deliberately not given an arbitrary gauge scale.

## Visual semantics

- filled semantic bands are shown only when canonical evidence exists;
- cyan band = target window;
- thin cyan marker = setpoint/reference;
- red marker/region = protection significance;
- current marker remains visually distinct from semantic bands;
- off-scale values remain numerically visible and are explicitly marked rather than silently clamped as normal;
- unavailable data remains unavailable;
- red is not used merely because a process fluid is hot.

## Explicit non-scope

M10.9.2 does **not** add:

- plant-mimic topology or animated pipes (M10.9.3);
- subsystem engineering schematics (M10.9.4);
- command consequence/dependency chains (M10.9.5);
- grid-demand challenge/scoring owners (M10.9.6/M10.9.7);
- new physics, controller, alarm or protection thresholds.

## Automated acceptance

Coverage must verify:

- deterministic logical-step trend direction/rate and reset-safe invalidation;
- independent scale/band/target/setpoint/protection semantics;
- explicit off-scale status that does not mutate the authoritative visual state;
- cold-shutdown presentation publishes scale/provenance metadata for the selected canonical instruments;
- turbine speed exposes the canonical overspeed protection marker;
- synchronization gauges expose canonical target windows;
- MainWindow uses both advanced linear and circular gauges in the intended workspaces;
- total turbine shaft power remains numeric until a canonical scale exists;
- the HMI legend states that on-scale is not equivalent to safe;
- all M10.9.1 shell and prior regression tests remain green.
