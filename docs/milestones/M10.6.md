## Goal

Add a deterministic M5-owned supervisory coordinator above the already validated local controller loops without creating a second physics/control owner.

## Delivered capability

`SupervisoryOperationCoordinator` accepts only bounded semantic objectives:

- hold reactor power at an explicit measured-unit setpoint;
- hold turbine speed at an explicit setpoint;
- hold the current measured reactor-power + turbine-speed operating point.

The coordinator translates those objectives only into existing canonical M5 controller modes/setpoints. It never assigns reactor power, pressure, level, rotor speed, generator output, breaker state or actuator state directly.

The operator computer MODES page exposes:

- independent training-assistance selection;
- `MANUAL`, `ASSISTED`, `SUPERVISORY` plant-control authority;
- `HOLD CURRENT OPERATING POINT` as the first structured supervisory objective available directly from the terminal;
- requested/effective authority, health/degradation, objective, transition sequence and per-loop controller modes.

The Application API also supports explicit bounded hold-power and hold-speed requests for deterministic orchestration/tests; arbitrary free-text or NLP commands remain forbidden.

## Safety/control invariants

- required feedback uses `MeasuredSignalFrame` only;
- invalid/unavailable required measurements degrade fail-closed to effective Assisted operation; no true-state fallback;
- active reactor SCRAM, turbine trip or generator trip suspends new supervisory decisions and protection remains latched/authoritative;
- no automatic SCRAM/trip reset or alarm acknowledgement;
- manual takeover stops supervisory decisions and converts local loops to bumpless manual outputs using committed controller state;
- same committed state + same objective/intents + same timestep produces the same supervisory decisions.

## Recorder/replay

M10.6 authority/objective intents are version-independent semantic replay inputs stored separately from physical `ControlRoomCommand` actions. `ScenarioFullReplayRunner` and checkpoint seek reapply the applicable intent prefix at the next-step boundary. Existing M9.1 `ControlRoomSnapshot` fingerprint v1 is preserved.

## Deliberate limits

- not a cold-start-to-grid autopilot;
- no autonomous startup/shutdown procedure executor;
- no automatic fault diagnosis using privileged M8 fault IDs;
- no direct supervisory load-following/generator-breaker sequencing yet;
- no change to physical solver ownership;
- no validation claim until the complete accumulated M10.2–M10.6 chain passes locally.
