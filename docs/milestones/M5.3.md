# M5.3 — Reactor & Primary-System Control Loops

## Goal

Bind the validated M5.1 measured-signal boundary and M5.2 controller/actuator primitives to concrete reactor and primary-system owners, while reusing existing M2 neutronics/rod physics and M3/M4 plant ownership.

## Delivered

- semantic `ReactorPrimaryControlLoopDefinition` / `ReactorPrimaryControlSystemDefinition` bindings;
- measured reactor-thermal-power to rod/group regulation loop;
- existing M2 control-rod motion and reactivity solvers reused without parallel rod state;
- existing M2 point kinetics and fission-power scaling composed into the controlled full-plant path;
- explicit non-rod-reactivity seam for temperature/void/xenon/manual contributions;
- canonical main-circulation flow and header-pressure-rise instrument sources;
- MCP flow/pressure support loops bound only to canonical main-circulation pumps;
- pump commands applied by replacing canonical operational `PumpState` before the one M4.7 physical step;
- effective M3 fission-power seam rewritten from point kinetics without duplicating spatial heat deposition;
- immutable reactor/primary loop diagnostics and controlled full-plant snapshot composition;
- verification for rod motion, reactivity progression, point-kinetics/fission-power coupling, pump command application and source/topology validation;
- ADR 0041 and reactor/primary control architecture documentation.

## Important ordering

Committed rod positions determine reactivity for the current point-kinetics step. Controller commands advance rods for the next committed state. This preserves committed-state determinism and avoids an implicit algebraic controller/reactor solve.

## Intentionally deferred

M5.4 owns turbine speed/load, steam/admission, drum/feedwater level and condenser/feedwater auxiliary control loops. M5.5 owns trips, SCRAM, permissives and interlocks.
