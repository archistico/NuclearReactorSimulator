# M7.2 — Cold Shutdown & Pre-Startup

## Objective

Provide the first concrete, versioned operational plant session on top of the validated M7.1 scenario framework: a deterministic cold-shutdown/pre-start initial condition plus observational readiness checks and guided preparation that stops before first criticality.

## Delivered

1. Exact built-in initial-condition identity `cold-shutdown-pre-start` version `1`.
2. `ColdShutdownInitialConditionFactory` reconstructing a fresh canonical M1–M5 runtime object graph through validated definitions, solvers and the simplified water/steam thermodynamic closure.
3. Cold/subcritical operational seed with modeled neutron/fission baseline at zero, rods inserted, main/condensate/feedwater pumps stopped, steam-admission valves closed, turbine stationary and generator breaker open.
4. Scenario metadata/objectives and fail-closed allowed-action set for shutdown verification and main-circulation preparation.
5. Explicit exclusion of control-rod withdrawal and generator-breaker closure so first criticality/synchronization cannot be smuggled into M7.2.
6. `PreStartupChecklistEvaluator` using only immutable `ControlRoomSnapshot` presentation state for instrumentation, protection, shutdown, rod, pump, turbine, breaker, steam-isolation and annunciator checks.
7. `PreStartupGuidancePlan` and ordered `PreStartupPreparationStepDefinition` records with optional suggested operator actions but no automatic dispatch or physical-state forcing.
8. Desktop composition loading the exact M7.2 session in `Paused` state through the validated M7.1 registry/session boundary instead of using the no-session shell as its default.
9. UI overview presentation of guided preparation and readiness observations; disallowed scenario commands fail closed and are surfaced as operator status rather than crashing the UI.
10. Application/runtime-composition tests for scenario permissions, guidance ordering, readiness evaluation, fresh runtime reconstruction, observable cold-shutdown conditions and MCP preparation.
11. ADR 0054 and `docs/COLD_SHUTDOWN_PRESTART.md`.

## Non-goals

- no rod-withdrawal or approach-to-criticality procedure — M7.3 owns first criticality;
- no low-power stabilization, period/reactivity/xenon training objectives;
- no primary heat-up, steam raising or turbine startup;
- no generator synchronization or breaker closure;
- no automatic procedure executor: guidance never dispatches operator commands;
- no arbitrary full-state checkpoint/save/seek format;
- no fault scheduling or hidden randomness;
- no scenario-side or UI-side duplicate physics/state owner.
