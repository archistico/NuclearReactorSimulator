## Goal

Activate the GUIDANCE, INFO and DIAGNOSTICS pages of the operator computer by projecting existing canonical M7 guidance/checklist owners and already-published M6 control-room values through generic immutable Application contracts.

## Status

**VALIDATED.**

M10.2 was validated as part of the cumulative M10.2→M10.6 Hotfix 1 clean build/test gate.

## Delivered scope

### Generic terminal content contracts

- `OperatorComputerInformationSnapshot` / sections / items;
- explicit information provenance: Measured / ModelDiagnostic / CanonicalState / Unavailable;
- `OperatorComputerGuidanceSnapshot` and generic step states;
- `OperatorComputerDiagnosticsSnapshot` and generic diagnostic items;
- `OperatorComputerScenarioContentSnapshot` composition envelope.

### Canonical projectors

- `OperatorComputerInformationProjector` reads only `ControlRoomSnapshot` and existing M6 panel snapshots;
- `OperatorComputerScenarioContentProjector` adapts the five existing M7.2–M7.6 guidance/checklist families without reimplementing their criteria and can project existing M7.7 training checkpoint/score assessment read-only;
- `OperatorComputerSnapshotProjector` marks INFO available universally, GUIDANCE/DIAGNOSTICS available only when canonical scenario content is supplied, and leaves future pages staged.

### App presentation

- terminal GUIDANCE page renders procedure/mode/current-step text;
- INFO renders compact REACTOR / PRIMARY / TURBINE / ELECTRICAL / PROTECTION sections with provenance markers;
- DIAGNOSTICS renders canonical evaluator results with satisfied/not-satisfied counts and observations, without inventing a universal readiness flag;
- existing F1–F8 fixed navigation remains unchanged;
- no free-form prompt or plant control is added.

## Ownership rules

M10.2 must not:

- compute reactor/plant physics;
- read private Simulation state;
- duplicate scenario checklist/readiness criteria;
- reinterpret unavailable values as zero;
- change training scoring/evaluation semantics;
- become runtime permissive/interlock authority.

## Tests

M10.2 adds/updates regressions for:

- unavailable information stays explicitly unavailable and is never fabricated as zero;
- fixed information section order/provenance;
- power-manoeuvring guidance/diagnostics reuse the canonical checklist evaluator;
- Guided vs Hidden/ChecklistOnly suppression semantics;
- page content-state staging (INFO available, missing guidance/diagnostics unavailable, future pages shell-only);
- real desktop scenario → `MainWindowViewModel` → terminal GUIDANCE/INFO/DIAGNOSTICS integration;
- XAML terminal content binding while preserving the fixed eight-page menu and no free-form input.

## Deliberate non-goals

ALARMS/LOG remain M10.3, COMMANDS M10.4, MODES/control-authority M10.5–M10.6 and SESSION persistence/replay workspace M10.7.
