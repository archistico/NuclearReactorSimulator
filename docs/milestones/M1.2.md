# M1.2 — Fluid Node Model

## Status

Validated via M1.2.1 — local build and complete test suite passed.

## Goal

Introduce the first immutable physical plant primitive: a deterministic lumped fluid control volume with conserved mass/internal energy, derived density/specific energy, and a clean seam for future thermodynamic closure.

## Delivered

### Domain fluid model

Added `Domain.Physics.Fluids`:

- `FluidNodeDefinition`;
- `FluidNodeInventory`;
- `FluidThermodynamicState`;
- `FluidNodeState`.

The model separates fixed geometry, conserved inventory and intensive thermodynamic variables.

### Conservation integration

Added `Simulation.Physics.Fluids`:

- `FluidNodeBalance`;
- `FluidNodeIntegrator`;
- `IFluidThermodynamicModel`;
- `FluidNodeDepletionException`.

The integrator applies signed net mass/energy rates over an explicit deterministic interval, creates a candidate conserved inventory, resolves thermodynamic closure, and returns a new immutable node state.

### Safety and consistency

- control volume must be > 0;
- populated nodes require mass > 0;
- populated nodes require pressure > vacuum;
- populated nodes require temperature > 0 K;
- density is always derived from mass/volume;
- specific internal energy is always derived from internal energy/mass;
- depletion to zero/negative mass fails before thermodynamic resolution;
- the input node state is never mutated.

### Verification

Tests cover:

- invalid geometry and identifiers;
- conserved/derived state relationships;
- thermodynamic absolute-state constraints;
- signed balance composition;
- mass and energy accumulation/removal;
- depletion fail-fast behavior;
- deterministic repeated integration;
- integration with the generic M0 fixed-step runtime and irregular external pulse segmentation.

## Explicitly out of scope

M1.2 does not introduce:

- a production water/steam equation of state;
- pipes or network flow;
- valves or pumps;
- heat transfer;
- phase equilibrium;
- enthalpy transport;
- vacuum/boundary reservoirs;
- reactor physics.
