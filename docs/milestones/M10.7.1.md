# M10.7.1 — Operator Control-State & Synchronization Usability Hotfix

Status: **IMPLEMENTATION CANDIDATE** on the validated M10.7 baseline.

## Purpose

Close operator-facing ambiguities found during manual use before M10.8:

- latched SCRAM / turbine-trip / generator-trip states must be visually persistent and must not look like re-clickable toggles;
- protection reset must be visible near the affected protection panels and F4 COMMANDS must report the same canonical M5.5 readiness/blocking reasons;
- synchronization indication must apply only to the pre-close breaker decision; a closed breaker is presented as `PARALLELED`, not as a persistent synchronization warning;
- the Overview must explain the current operating condition, the next canonical action and the command path from cold shutdown to first electrical output.

## Architecture constraints

- trip commands remain one-shot protection intents; they do not become toggles;
- M5.5 remains the sole protection/reset owner;
- M4.5 remains the sole synchronization/breaker owner;
- reset readiness and synchronization diagnostics are presentation-only projections and do not alter physical state;
- new presentation-only diagnostic fields are excluded from `ControlRoomSnapshotFingerprint` v1;
- guidance composes existing validated M7 procedure/checklist metadata and dispatches nothing automatically.
