## Goal

Refine the validated M3.3 aggregated-core projection without replacing the validated global M2 point-kinetics owner.

M9.4 adds an optional quasi-spatial layer that:

- evaluates existing linear fuel-temperature, coolant-temperature and void-reactivity feedback formulas on committed per-zone plant domains;
- reduces those local values to one deterministic power-weighted scalar reactivity contribution for the existing global point-kinetics seam;
- supports explicit symmetric zone-to-zone coupling for smoothing only the power-shape driving signal;
- evolves normalized zone power shares deterministically toward a feedback-biased target with an explicit relaxation time;
- works with arbitrary configured aggregated-core zone counts/coordinates and therefore supports higher-resolution aggregations without embedding a fixed grid.

## Feedback weighting

For each configured zone, M9.4 reads the canonical committed:

- fuel thermal-body temperature;
- coolant fluid-node temperature;
- coolant thermodynamic state/void fraction.

The existing validated linear M2 feedback equations are reused. M9.4 does not create replacement temperature/void physics.

When the M9.4 profile is enabled, callers must not also pre-add those same configured fuel/coolant/void contributions into `ReactorPrimaryControlInputs.NonRodReactivity`; that scalar seam remains available only for distinct externally composed contributions. This avoids double-counting one physical feedback owner.

The scalar reactivity contribution consumed by global point kinetics is the deterministic current-power-share-weighted sum of local zone feedback values.

## Zone coupling

Coupling is explicit configuration data, not inferred from grid adjacency.

`CoreZoneCoordinate` therefore remains logical placement metadata and arbitrary/non-rectangular layouts remain supported.

Each symmetric `CoreZoneCouplingDefinition` specifies the fraction of one zone's local shape-driving signal blended with its neighbour. Incident coupling fractions for a zone may not exceed 1.0.

Coupling affects only power-shape redistribution. It is **not neutron diffusion**, does not transfer neutrons, and does not create local kinetics states.

## Power-shape refinement

The solver compares each coupled zone signal with the current power-weighted global feedback and converts the deviation into a positive normalized target weighting through the configured `CorePowerShapeSensitivity`.

The committed shape then relaxes deterministically toward that target using the explicit `PowerShapeRelaxationTime`.

Properties:

- every zone share remains within `[0, 1]`;
- shares remain normalized to 1.0;
- no local neutron population is stored;
- no mass/energy inventory is duplicated;
- fission-power closure remains owned by `AggregatedCorePowerSolver` / `FuelChannelGroupSolver` and the canonical plant-network integration path.

## Optional higher-resolution aggregation

M9.4 does not force a new built-in full-plant topology or silently reinterpret existing versioned scenario seeds.

The refinement is defined against the already configurable `AggregatedCoreDefinition` and is tested with non-fixed and higher zone counts. A future explicitly versioned plant/initial-condition configuration may opt into more zones and matching channel-group topology without changing the M9.4 solver or global point-kinetics ownership.

This avoids silently changing M7/M9.1 exact-version replay semantics.

## Compatibility rule

When `ReactorPrimaryControlSystemDefinition.QuasiSpatialCoreFeedbackDefinition` is `null`:

- no spatial feedback contribution is added;
- no quasi-spatial solver is invoked;
- existing `IntegratedPrimaryCircuitInputs.CoreState` remains the full-plant power-shape input exactly as before;
- legacy scenario/initial-condition behavior is not silently upgraded.

M9.4 is therefore opt-in like M9.3 xenon promotion.

## Deliberate limitations

M9.4 does **not** add:

- multi-node neutron diffusion or transport;
- independent per-zone neutron populations or delayed-neutron inventories;
- local iodine/xenon inventories or spatial xenon oscillations;
- channel-by-channel full-core physics;
- inferred geometric coupling from coordinates;
- hidden nonlinear same-step iteration;
- licensing-grade spatial neutronics;
- automatic modification of existing versioned M7/M8/M9.3 initial conditions.

Those capabilities require separate explicit ownership, calibration and validation.
