# M5.7 — Integrated Automatic-Operation Baseline

Status: **VALIDATED**

## Scope

- canonical M5.1–M5.6 automatic-operation state/input/snapshot composition;
- committed measured-frame ordering across repeated logical steps;
- instrumentation publication for the next step only after the one protected physical evolution;
- headless deterministic verification phases for reference hold, setpoint changes, explicit disturbances and protection/interlock matrix cases;
- measured-signal tracking criteria and raw full-plant mass/energy closure criteria;
- immutable verification results with no hidden state correction.

## Acceptance criteria

1. Normal control, protection and alarms consume the same committed measured frame for each logical step.
2. Candidate true-state measurements are published only for the next logical step.
3. The full physical plant evolves exactly once per M5.7 step.
4. Repeated execution from identical state/inputs is deterministic across physical, controller, protection, alarm and instrumentation owners.
5. Verification phases can change explicit setpoints/plant inputs without adding a hidden scenario scheduler.
6. Protection/interlock expectation cases are checked deterministically through the same integrated runtime path.
7. Tracking criteria fail on invalid/unavailable measurements and never fall back to true state.
8. Verification criteria are observational and never correct state to force a pass.
9. Full mass and reactor-to-grid energy closure residuals remain visible and bounded by explicit criteria.
