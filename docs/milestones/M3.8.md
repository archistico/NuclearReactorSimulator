# M3.8 — Integrated Primary-Circuit Baseline

## Status

VALIDATED — local build and complete automated test suite passed.

Built on validated M3.7 — Feedwater & Steam Boundary Interfaces.

## Delivered

- canonical `IntegratedPrimaryCircuitDefinition` spanning the complete M3.3–M3.7 semantic lineage;
- immutable `IntegratedPrimaryCircuitInputs` for core state, fission power, decay heat and temporary external boundaries;
- `IntegratedPrimaryCircuitSolver` that solves every subsystem from one committed plant state;
- deterministic composition of channel nuclear heat, steam-drum separation and feedwater/steam boundary source terms;
- exactly one `PlantNetworkOrchestrator` integration of conserved inventories per integrated step;
- immutable `IntegratedPrimaryCircuitSnapshot` with global and per-subsystem diagnostics;
- total plant water/steam mass, stored energy, nuclear power, feedwater and steam-export aggregates;
- configurable `PrimaryCircuitReferenceOperatingPoint` without hidden control/correction logic;
- headless `PrimaryCircuitLongRunRunner` and immutable drift/residual summary;
- deterministic long-run zero-net-source equilibrium regression;
- integrated conservation/audit verification and canonical-definition tests;
- ADR 0031 and `docs/INTEGRATED_PRIMARY_CIRCUIT.md`.

## Deliberately not included

- turbine island or main steam-line dynamics;
- condenser/hotwell/closed secondary-cycle return;
- automatic process control;
- trips/interlocks/protection;
- production control-room UI;
- forced steady-state correction or hidden mass/energy balancing.

## M3 gate

M3.8 is locally validated, so M3 is complete. The primary circuit has a canonical headless integration boundary, deterministic long-run verification, explicit conservation audits and a plant-level snapshot suitable for later instrumentation.

The next roadmap milestone is **M4.1 — Main Steam Network & Turbine Admission**, now implemented as the current baseline candidate.
