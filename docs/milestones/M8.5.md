# M8.5 — Educational Leak/LOCA-Class Scenarios

**Status:** VALIDATED / HOTFIX 2  
**Prerequisite baseline:** M8.4 hotfix 2 — VALIDATED

## Goal

Add deterministic educational leak/LOCA-class scenarios that create real conserved inventory-loss and depressurization consequences without claiming licensing-grade accident-analysis fidelity or introducing a second thermal-hydraulic owner.

## Delivered scope

- `loca.pressure-driven-break` fault type with exact M8.1 lifecycle/scheduling.
- Bounded pressure-driven mass-flow law derived from committed source-node pressure and an explicit ambient/reference pressure boundary.
- Signed mass and carried-internal-energy removal through the existing `PlantNetworkSourceTerms` / single `PlantNetworkOrchestrator` integration boundary.
- Per-step inventory-fraction guard plus deterministic thermodynamic-admissibility cap: the declared fraction remains the maximum requested removal, while the actual break source is reduced further when necessary to keep the candidate node inside the validated simplified water/steam envelope.
- Three deterministic scenario definitions:
  - small primary-coolant leak;
  - large break-class loss-of-coolant challenge;
  - steam-space leak/depressurization.
- Fail-closed exact node binding and prevention of silently stacking a prescribed M8.2 leak with an M8.5 pressure-driven break on the same node.
- Regression tests for mass/energy loss, relative depressurization, zero flow without positive driving pressure, declared inventory-removal bound, thermodynamic-envelope-safe severe break requests, built-in scenario binding and invalid target handling.

## Ownership rules

1. M3 remains the sole owner/integrator of conserved fluid/thermal inventories.
2. M8.5 computes only an external signed source term from committed state; it never writes node pressure, temperature, void or level directly.
3. Break flow uses committed source-node pressure and deterministic scenario parameters only; no wall clock, randomness or hidden accident trajectory is allowed.
4. Fault clearance removes the break forcing only. Lost inventory/energy is never restored automatically.
5. Protection, controller and operator responses remain owned by M5/M7 and observe the consequences through their existing measured-signal/runtime seams.

## Explicit fidelity limits

The M8.5 break law is intentionally educational. It is **not** a critical-flow, choked two-phase discharge, flashing-jet, pipe-rupture mechanics, containment, ECCS, fuel-damage or licensing-grade LOCA model. The square-root pressure relation is bounded by a declared reference flow and per-step inventory fraction. Scenario results are meaningful only within the validated lumped water/steam envelope of this simulator.
