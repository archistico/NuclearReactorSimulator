# M1.2.1 — Thermodynamic Closure Test Hotfix

## Status

Validated — local build and complete test suite passed.

## Context

The first local M1.2 test run reported one failure in `FluidNodeIntegratorTests.ZeroBalance_PreservesConservedInventory`. The production integrator behaved according to ADR 0009: after integrating the conserved inventory, it invoked `IFluidThermodynamicModel.Resolve` once to obtain pressure and temperature.

The test was internally contradictory: it asserted both that the thermodynamic model had been invoked once and that the previous thermodynamic state had been preserved, even though the configured recording model deliberately returned a different pressure and temperature.

## Change

The regression test now verifies the intended contract:

- zero net mass/energy rates preserve mass and internal energy;
- thermodynamic closure is still invoked once;
- the resulting pressure/temperature equal the state returned by the configured thermodynamic model.

No production code, API, conservation rule or runtime behavior changed.
