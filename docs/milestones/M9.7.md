The first M9.7 validation run found four concrete integration defects and the stale descriptor regression:

- the descriptor test still expected M9.6 although the composed descriptor correctly reported M9.7;
- the default desktop M7.7-derived low-load seed could drive `control-out` outside the simplified water/steam envelope after only a few manual steps;
- `RUN` changed host mode but Avalonia had no periodic owner calling `ControlRoomRuntimeCoordinator.AdvanceRunning`;
- operational push buttons did not present a clear pointer/full-surface hit target;
- the footer/status bar visually obscured the bottom of long center-workspace content.

Hotfix 1 resolves these without moving physics into App:

- the validated M7 v1 initial-condition/scenario identities remain untouched; the desktop now uses a new explicit M9.7 versioned integration seed/profile with a warm turbine steam-path inventory and governor droop/opening, plus a 100-step endurance regression;
- `DesktopControlRoomRuntimePump` is an App-layer cooperative host driven by `DispatcherTimer` at 50 ms. Each tick requests five existing 10 ms deterministic fixed steps and publishes only the final snapshot. UI cadence therefore schedules execution but does not alter timestep physics;
- a deterministic step failure pauses host RUN and reports the failure instead of producing a repeated timer exception storm;
- the header/footer expose runtime state and logical-step progress;
- `ControlRoomPushButton` uses a full non-null surface, stretch alignment and `Hand` cursor;
- the center row/scroll viewport is clipped above the footer with explicit bottom scroll space.

## Hotfix 4 — thermodynamic closure finding from 60-second endurance

Hotfix 3's longer tests exposed `WaterSteamStateOutOfRangeException` at both ends of the saturation envelope (`drum` near quality 0 and `exhaust` near high-quality wet steam). Direct inspection confirmed that the conserved states are supported by the existing simplified correlations; the original 512-sample full-range bracket scan can simply miss the root when the admissible specific-volume interval terminates between samples.

Hotfix 4 therefore changes numerical bracketing, not the physical property model: the existing saturated/subcooled/superheated fast paths remain first. Only before final out-of-range failure, a boundary-aware saturated-mixture search determines the exact upper admissible saturation temperature for the node specific volume and rescans that valid interval with the same deterministic bisection. Exact regressions cover both observed failure states. No arbitrary tolerance-based phase clamp, no enlarged state envelope and no new thermodynamic correlation are introduced.

The desktop-specific inflated liquid-compression override is no longer used as the stability mechanism; the versioned desktop profile retains only its balanced low-load steam/turbine/condenser lineup.

## Hotfix 5 — superheated-onset root bracketing

Hotfix-4 revalidation exposed a later `exhaust` state at `v=65.477888248812704 m^3/kg`, `u=2434381.9782870663 J/kg`. The state is not outside the existing model: the current superheated equations contain a root at about 17.907 °C / 2.052 kPa, but the admissible superheated interval starts between two coarse scan samples and the first coarse valid point is already beyond the root.

Hotfix 5 keeps every existing resolver branch/equation intact and adds only a final boundary-aware superheated rescan. It identifies the exact contiguous admissible superheated-temperature interval for the fixed specific volume, includes its endpoints and reuses the existing residual/bisection. A negative regression confirms that states in a true correlation gap with no root still throw `WaterSteamStateOutOfRangeException`; no envelope widening or phase clamp is introduced. The 6,000-step / 60-second gates remain mandatory.
