# M6.6 — Trends, Alarms & Event Timeline

## Objective

Complete the historical/annunciator presentation layer of the M6 control room using only immutable presentation snapshots and validated M5.6 alarm semantics.

## Delivered

1. `AlarmEventsPanelSnapshot` plus presentation-only alarm, first-out and event records.
2. Projection of M5.6 severity, annunciator state, latching, acknowledgement, first-out and logical event sequence into the Application boundary.
3. Typed targeted/bulk alarm ACK and RESET operator command intents.
4. Stable configurable `ControlRoomTrendSourceCatalog` over presentation values only.
5. Bounded `ControlRoomOperationalHistoryAccumulator` indexed by logical step and deduplicated alarm sequence.
6. Deterministic textual sparklines with explicit unavailable gaps and min/max/current diagnostics.
7. Production `Alarms & Events` workspace containing trends, annunciator, first-out groups, alarm controls and event timeline.
8. Tests for bounded deterministic history, trend catalog constraints, typed alarm commands and presentation-type dependency safety.
9. ADR 0051 and `docs/TRENDS_ALARMS_EVENT_TIMELINE.md`.

## Non-goals

- no wall-clock historian;
- no disk persistence;
- no UI-side alarm/protection re-evaluation;
- no reset of M5.5 protection from alarm ACK/RESET;
- no complete live runtime command translation (M6.7).

## Validation status

Validated locally by the user: compilation and complete test suite passed. M6.6 is the validated baseline for M6.7.
