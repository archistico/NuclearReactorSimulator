# M10.4 — Contextual Command Console

## Validation result

The user later confirmed the cumulative M10.2→M10.6 Hotfix 1 package compiled successfully and the complete automated suite passed. This milestone is therefore VALIDATED as part of that cumulative gate.

## Status

**VALIDATED.**

## Delivered scope

- immutable Application-layer command-console contracts;
- contextual projection of canonical `ControlRoomCommandKind` commands and exact targets;
- deterministic command grouping, state, availability and explicit block reasons;
- `COMMANDS` fixed terminal page activated;
- keyboard/list selection plus Enter/explicit execute action;
- dispatch only through `IControlRoomCommandDispatcher`;
- runtime/scenario rejection remains authoritative and visible to the operator;
- no direct Simulation writes and no second permissive/interlock owner.

## Key semantics

- `AVAILABLE` is advisory and means only that the presentation snapshot exposes no known blocker;
- `BLOCKED` uses already-published state to explain a definite presentation-level blocker;
- `UNAVAILABLE` means the runtime or required target is absent;
- every dispatched command is revalidated fail-closed by the existing canonical boundary;
- per-target commands preserve canonical `TargetId` and `ControlRoomCommandTargetKind` exactly;
- alarm acknowledge/reset never resets protection.

## Deliberate omissions

- MODES / training-assistance mutation — M10.5;
- real Manual / Assisted / Supervisory authority — M10.5–M10.6;
- supervisory automation — M10.6;
- session/checkpoint/replay/save lifecycle — M10.7;
- no free-form textual command parser.
