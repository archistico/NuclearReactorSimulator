# M8.2 — Hydraulic Component Faults

**Status:** BASELINE CANDIDATE

## Scope

- concrete deterministic fault applicators for main/auxiliary pump trip and capacity degradation;
- valve fail-open, fail-closed and stuck-at-activation-position semantics;
- deterministic restriction/blockage of canonical valve-controlled hydraulic paths;
- selected fluid-node leaks represented as explicit signed mass/energy source terms;
- runtime-bound fault-effect target that converts M8.1 declarations into immutable per-step hydraulic fault inputs;
- effects applied after normal/protection command arbitration but before the one authoritative physical step, so failed equipment may be unable to follow a valid protection/control command without creating a second owner;
- leak mass and carried internal energy removed through the existing `PlantNetworkSourceTerms` / `PlantNetworkOrchestrator` boundary;
- built-in M8.2 hydraulic applicator registration and a deterministic demonstration scenario pack.

## Acceptance

- pump trip forces zero effective hydraulic speed while active, even if normal control requests operation;
- pump degradation scales the commanded effective speed deterministically without creating another pump solver;
- valve fail-open/fail-closed/stuck and path restriction/blockage constrain canonical `ValveState` before the existing flow solvers run;
- valve-stuck captures only the committed mechanical position at activation and holds that position while active;
- selected leak effects remove mass and the source node's carried internal energy in the same single network integration, with signed external mass/power audit terms;
- unknown canonical pump/valve/node targets and invalid parameters fail closed;
- no M8.2 applicator directly writes flow, pressure, temperature, reactor power, turbine output or electrical output;
- no wall-clock/random behavior is introduced;
- complete build and automated suite must pass locally before M8.2 validation.

## Deliberate limits

M8.2 path restriction/blockage is limited to canonical **valve-controlled paths**. Arbitrary raw-pipe rupture/resistance mutation and larger break/LOCA behavior remain M8.5 ownership. M8.2 selected leaks use bounded prescribed mass-flow removal suitable for the existing simplified thermohydraulic envelope; they are not licensing-grade break-flow models. Instrumentation/control faults remain M8.3 and turbine/generator/feedwater/condenser transient packs remain M8.4.
