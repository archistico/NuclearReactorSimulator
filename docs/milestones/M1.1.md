# M1.1 — Physical Quantities & Units

## Status

Validated through M1.1.1 local build/test suite.

## Goal

Establish a deterministic, dependency-free and strongly typed physical unit boundary before introducing the first fluid node.

## Delivered

### Strong physical value types

Implemented in `NuclearReactorSimulator.Domain.Physics.Quantities`:

- `Length`, `Area`, `Volume`;
- `Mass`, `Density`;
- `Temperature`, `TemperatureDifference`;
- `Pressure`, `PressureDifference`;
- `Energy`, `SpecificEnergy`, `Power`;
- `MassFlowRate`, `VolumetricFlowRate`.

### Safety rules

- canonical SI storage;
- explicit non-SI factories/conversions;
- no implicit `double` conversions;
- all values reject `NaN` and infinities;
- intrinsically absolute quantities reject negative values;
- temperature and pressure have distinct signed difference types;
- invalid derived operations (for example density from zero volume) fail explicitly.

### Dimensional operations

Added a deliberately small set of dimensionally meaningful relationships needed by upcoming plant primitives, including geometry, density, energy/power and pressure-volume work.

### Automated verification

Domain tests cover:

- construction constraints;
- non-finite rejection;
- SI/non-SI conversion accuracy;
- absolute-vs-difference semantics;
- dimensional arithmetic;
- divide-by-zero and invalid-duration behavior;
- value equality and comparisons;
- valid zero defaults for all quantity structs.

## Explicitly out of scope

M1.1 does not introduce:

- fluid nodes;
- equations of state;
- water/steam property tables;
- viscosity or friction models;
- pumps, valves or pipes;
- heat-transfer coefficients;
- enthalpy/entropy-specific semantic state models;
- reactor or neutron physics.

Those quantities and semantics will be introduced only when the corresponding model requires them.
