# M3.1 — Plant Composition & Topology Baseline

## Status

VALIDATED

## Objective

Create the first immutable plant-level composition boundary for already validated physical primitives, without adding or changing any physical equation.

## Delivered

### Domain

- `PlantDefinition` with canonical registries for fluid nodes, pipes, valves, pumps, thermal bodies, heat-transfer links and heat sources.
- Global topology-ID uniqueness, including nested valve/pump hydraulic-path definitions.
- Eager topology validation for every hydraulic endpoint and thermal-domain reference.
- `PlantState` with exact one-to-one ownership of all stateful component states.
- Canonical definition consistency checks for fluid-node and thermal-body states.
- Deterministic typed lookup methods.

### Simulation boundary

- `PlantSnapshot` as the first immutable plant-level committed-state observation boundary.
- No solver execution or new physical behavior in the snapshot layer.

### Tests

- registry canonicalization and caller-array independence;
- missing hydraulic endpoint rejection;
- missing thermal-domain/heat-source target rejection;
- global topology-ID collision rejection;
- exact state-set validation;
- canonical-definition mismatch rejection;
- plant-state canonical ordering;
- plant-snapshot lookup and component-kind isolation.

### Documentation

- `docs/PLANT_COMPOSITION.md`;
- ADR 0024;
- roadmap, architecture, project status, README and changelog updates.

## Explicit non-goals

- no multi-component solver orchestration;
- no balance accumulation;
- no network iteration;
- no core-zone/channel model;
- no RBMK constants;
- no steam-drum physics.

## Acceptance

M3.1 was validated by successful local build/test execution. Topology/state tests confirm canonical, complete and fail-fast plant composition.
