# M2.1 — Reactivity Model

## Goal

Introduce signed reactor reactivity as a strongly typed physical primitive and establish deterministic, diagnostic composition of independent reactivity contributions without implementing neutron kinetics yet.

## Delivered

- `Reactivity` with canonical `delta-k/k` storage and explicit percent/pcm conversions;
- signed arithmetic and finite-value validation;
- named immutable `ReactivityContribution` values;
- diagnostic categories for control rods, fuel/coolant temperature, void, xenon and other effects;
- deterministic canonical contribution ordering;
- compensated summation and per-category subtotals;
- immutable `ReactivityBreakdownSnapshot`;
- duplicate-contribution identity validation;
- deterministic runtime integration test;
- ADR 0015 and reactivity-model documentation.

## Explicitly deferred

- control-rod worth and motion;
- neutron point kinetics and delayed-neutron groups;
- reactor period;
- direct thermal-power generation;
- temperature/void feedback equations;
- iodine/xenon inventory dynamics;
- dollars/cents reactivity units.

## Acceptance checklist

- solution restores successfully;
- solution builds with warnings treated as errors;
- all tests pass;
- equivalent reactivity units compare correctly;
- signed contributions compose to the expected total;
- input permutation does not alter canonical diagnostics or total;
- duplicate contribution IDs fail fast;
- external pulse segmentation does not alter runtime reactivity snapshots;
- no direct reactivity-to-power shortcut exists.
