# M5.5 — Interlocks, Trips & SCRAM

Status: **VALIDATED**

## Goal

Add deterministic measured-signal protection above the validated M5.3/M5.4 automatic-control stack, with explicit protection-over-control arbitration and no duplicate physical state.

## Delivered

- `ProtectionSystemDefinition` over the canonical M5.1 instrumentation and M4.7 full-plant definition;
- measured high/low latching trip functions with separate trip/reset thresholds and explicit invalid-measurement fail-closed policy;
- explicit reactor SCRAM, turbine trip and generator trip action flags;
- deterministic manual trip and reset command seams;
- measured reset permissives and observable accepted/rejected reset behavior;
- non-latching rod-withdrawal, turbine-admission-opening and generator-breaker-close interlocks;
- protection state containing only logical latches/manual trip memory;
- SCRAM arbitration through canonical M2 control-rod insert commands while preserving committed-state kinetics ordering;
- turbine trip arbitration through canonical M4.1 stop-valve closure plus the existing M4.2 `TripCommand` seam;
- generator trip arbitration through the existing M4.5 breaker-open seam;
- `ProtectedAutomaticFullPlantSolver` composing M5.3 + M5.4 + M5.5 over one measured frame and one M4.7 physical full-plant step;
- explicit protection and arbitration snapshots separate from alarm presentation;
- ADR 0043 and protection architecture documentation.

## Intentionally deferred

M5.6 owns alarm conditions, annunciator latching, acknowledgement/reset presentation and first-out/event semantics. M5.7 owns the integrated automatic-operation/protection baseline and test matrix.
