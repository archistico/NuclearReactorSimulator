# M1.5 — Pumps

## Goal

Introduce the first deterministic active hydraulic machine without breaking the validated pressure-driven network architecture.

## Included

- strongly typed normalized `PumpSpeed`;
- strongly typed `PumpEfficiency`;
- immutable `PumpDefinition` and `PumpState`;
- active pressure boost following the speed-squared affinity law;
- quadratic internal pump-curve resistance;
- signed bidirectional flow solved from endpoint pressure plus pump pressure source;
- volumetric-flow derivation from upstream density;
- signed hydraulic power exchange and positive non-regenerative shaft demand;
- mass-conservative and energy-accounted endpoint balances;
- stopped-pump passive-flow behavior;
- deterministic fixed-step integration tests.

## Explicitly excluded

- rotor inertia;
- motor/electrical dynamics;
- cavitation/NPSH;
- detailed pump maps;
- check valves;
- pump control/interlocks;
- loss-to-heat conversion;
- two-phase degradation.

## Acceptance

- `dotnet restore` succeeds;
- `dotnet build --no-restore` succeeds with warnings as errors;
- `dotnet test --no-build` passes;
- rated pump creates pressure-driven flow at equal node pressures;
- speed affinity behavior is verified;
- stopped pump produces no active boost or shaft demand;
- flow reverses under sufficient opposing pressure;
- total endpoint mass rate is exactly zero;
- total endpoint energy rate equals active hydraulic power exchange;
- fixed-step pulse segmentation produces identical final snapshots.
