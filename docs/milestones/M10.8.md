# M10.8 — Integrated Operator Computer UI

**Status:** VALIDATED
**Prerequisite baseline:** M10.7.1 Hotfix 3 — VALIDATED

The user explicitly confirmed local compilation and the complete automated suite passed. M10.8 is the official validated baseline for M10.9.1.

## Goal

Integrate the already validated M10.1–M10.7.1 operator-computer capabilities into one coherent Avalonia workstation without introducing new physics, control ownership, alarm ownership, replay ownership or free-form command semantics.

## Scope

- fixed terminal header, page menu, plant-status summary and footer;
- eight fixed pages: GUIDANCE, INFO, ALARMS, COMMANDS, MODES, DIAGNOSTICS, LOG, SESSION;
- persistent visual indication of the selected page;
- compact monospace/tabular HUD presentation;
- only page content scrolls; menu/status remain visible;
- keyboard-first operation: global F1–F8 page access, Tab/Shift+Tab focus traversal, arrow-key list navigation and Enter activation;
- full mouse operation remains available;
- COMMANDS and SESSION layouts remove rigid list heights/over-wide columns that compressed poorly in the validated control-room center viewport;
- no `TextBox`, free-form command prompt, NLP parser or external model dependency.

## Architecture boundary

M10.8 is presentation integration only.

- `OperatorComputerViewModel` exposes selection/status presentation properties only.
- Avalonia binds existing Application contracts and commands.
- no physical calculations, permissives, controller algorithms or replay decisions move into App/ViewModels;
- M2/M3/M4/M5/M7/M9/M10.5–M10.7 owners remain unchanged.

## Acceptance

Automated coverage must verify:

- exactly eight fixed pages remain present;
- exactly one page-selection presentation state is active at a time;
- fixed header/status/footer layout surrounds one scrollable page-content region;
- runtime, logical step, alarms, signal health and protection state remain visible outside the scrollable page body;
- COMMANDS remains keyboard executable through Enter and typed-command dispatch;
- SESSION retains replay-backed lifecycle controls;
- no free-form command surface is introduced;
- existing MainWindow global F1–F8 bindings remain intact.

Manual validation should follow `docs/M10_8_MANUAL_VALIDATION_CHECKLIST.md` and confirm:

1. F1–F8 opens/selects every page from any control-room workspace;
2. selected-page underline/indicator is unambiguous;
3. Tab/Shift+Tab reaches every interactive terminal control; arrows and Enter operate lists/buttons without a mouse;
4. status summary remains visible while scrolling long GUIDANCE/LOG/SESSION content;
5. COMMANDS and SESSION remain usable at the validated minimum window size without overlapping/clipped controls;
6. M10.7.1 trip/reset/synchronization/persistent-control feedback remains unchanged;
7. no page navigation or UI interaction changes physical state unless an explicit canonical command is dispatched.
