# M6.2 — Reusable Instrument & Control Components

Status: **VALIDATED**

## Scope

- shared semantic presentation states: Normal, Warning, Trip and Unavailable;
- reusable numeric indicators, meters and status lamps;
- reusable toggle switches, selectors and pushbuttons;
- stable keyboard/mouse interaction rules suitable for simulator operation;
- Application-layer component catalog without Avalonia coupling;
- M6 shell component gallery for visual validation.

## Acceptance criteria

1. The reusable library includes numeric indicator, meter, lamp, toggle switch, selector and pushbutton primitives.
2. All components use the same four semantic visual states.
3. Display-only components do not accept operator state changes.
4. Interactive controls use predictable focus/keyboard/pointer semantics and disable command acceptance when unavailable.
5. Operational pushbutton commands still leave Avalonia through Application command dispatch; no UI physics is introduced.
6. Application presentation contracts remain free of Avalonia types and App remains free of direct Simulation namespace references.
7. The M6 shell demonstrates the reusable component family without pretending to be a finished plant-specific panel.
8. Rendering/interaction cadence remains presentation-only and cannot affect deterministic simulation results.
9. Reactor/core panel composition remains deferred to M6.3.

## Validation result

Validated locally on 2026-07-21: build succeeded and the complete automated test suite passed. M6.2 is the baseline for M6.3.
