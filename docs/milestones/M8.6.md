# M8.6 — Electrical Loss & Station Blackout-Class Scenarios

**Status:** VALIDATED

## Goal

Add deterministic educational loss-of-external-supply and station-blackout-class scenarios using only canonical electrical, hydraulic, control and protection seams already present in the validated M1–M8.4 architecture.

M8.6 was originally produced as a stacked candidate over M8.5. The exact M8.5 hotfix 2 → M8.6 → M8.7 hotfix 2 chain subsequently passed local clean restore/build and the complete automated suite, so M8.6 is validated as part of the M8 gate.

## Scope

- `electrical.external-supply-loss` exact-grid fault type;
- fail-closed `IElectricalLossFaultTarget` binding;
- active external-supply loss forces canonical M4.5 generator breakers open through `GeneratorGridInputs` and overrides close requests without writing breaker state directly;
- fault clearance removes forcing only and never reconnects the generator automatically;
- deterministic `m86-external-supply-loss` scenario;
- deterministic `m86-station-blackout-class` scenario composed explicitly from M8.6 external supply loss plus validated M8.2 pump trips, M8.3 powered actuator-command loss and M8.4 turbine/generator trips;
- documentation of the integrated-runtime decay-heat gap instead of fabricating a scenario-owned decay source.

## Acceptance criteria

1. External supply loss opens the canonical generator breaker through the existing M4.5 input/solver path.
2. An operator breaker-close request cannot defeat an active external-supply-loss fault.
3. Clearing the fault does not automatically restore/reclose the breaker.
4. Unknown grid IDs fail closed.
5. Built-in M8.6 scenarios bind using the standard M8.1 registry with no custom scheduler.
6. Station-blackout-class pump/control consequences are explicit individual validated fault declarations rather than hidden side effects.
7. No App/Scenario code writes electrical output, torque, rotor speed, pump flow or physical breaker state directly.
8. No synthetic bus/diesel/battery/ECCS state owner is introduced.
9. No fixed-percentage or scenario-owned decay-heat model is introduced while M2.5 state is absent from the integrated M5.7 runtime envelope.
10. Local warnings-as-errors build and complete automated tests pass before validation.


## Validation record

Validated with M8.5 hotfix 2 and M8.7 hotfix 2 after local clean restore/build and complete tests passed. No separate station-bus/diesel/battery model was introduced.
