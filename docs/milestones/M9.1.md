# M9.1 — Recorder, Checkpoints & Full Replay

**Status:** VALIDATED
**Prerequisite baseline:** M8.7 hotfix 2 — VALIDATED / M8 gate COMPLETE

## Goal

Add deterministic session recording, versioned replay-backed checkpoints and full replay verification without creating a second physical-state owner or coupling replay correctness to wall clock/UI publication cadence.

## Delivered scope

- `ScenarioRecorder` attached to a fresh `ScenarioSession`:
  - captures the initial frame plus every `DeterministicStepCompleted` frame;
  - remains independent from `SnapshotChanged` publication stride;
  - records accepted operator actions from the existing M7.7 `ScenarioOperatorActionJournal`;
  - records alarm events, M8 fault lifecycle transitions and protection-trip transitions as one monotonic logical event stream.
- `ScenarioRecording` immutable session record containing:
  - exact scenario id and versioned initial-condition identity;
  - contiguous logical-step frames with retained `ControlRoomSnapshot` values;
  - accepted typed operator actions;
  - deterministic recorder events;
  - optional replay-backed checkpoints.
- `ControlRoomSnapshotFingerprint` v1:
  - SHA-256 deterministic fingerprint of the control-room snapshot;
  - host run/pause state normalized so execution mode/publication orchestration does not create false replay divergence.
- `ScenarioCheckpoint` schema v1:
  - exact scenario/initial-condition identity;
  - logical step;
  - last applied accepted-operator-action sequence;
  - versioned fingerprint algorithm id and expected snapshot fingerprint.
- `JsonScenarioCheckpointSerializer` with explicit schema-version rejection for unsupported versions.
- `ScenarioFullReplayRunner`:
  - reconstructs a fresh exact-version scenario session;
  - replays accepted operator actions at their next fixed-step application boundary;
  - reuses scenario-declared M8 fault scheduling rather than recording a second fault trace;
  - verifies every frame fingerprint fail-closed;
  - verifies the complete deterministic event stream;
  - supports replay-backed seek to a checkpoint and verifies the anchor before returning the session.
- Regression tests for publication-stride independence, operator-action step mapping, full frame/event replay, checkpoint seek, tamper divergence and checkpoint JSON versioning.

## Ownership rules

1. The recorder is observation/application state only; it never mutates simulation, control, protection or fault state.
2. Checkpoints are replay anchors, not authoritative physical-state dumps.
3. Replay always starts from an exact versioned initial condition and scenario definition.
4. Operator commands are replayed from accepted typed action history only; runtime host commands are not part of the physical command trace.
5. Scenario faults are reconstructed from scenario data and M8.1 lifecycle rules; M9.1 does not create a duplicate fault trace.
6. Replay divergence fails closed at the first mismatching logical-step fingerprint.
7. UI publication cadence and host run/pause state cannot change recorded deterministic step coverage or replay fingerprints.

## Non-goals

- no binary serialization/restore of internal solver object graphs;
- no random-access state mutation;
- no wall-clock timestamps;
- no post-incident causal analysis UI (M9.2);
- no model calibration/fidelity changes.
