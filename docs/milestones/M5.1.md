# M5.1 — Instrumentation & Signal Model

## Goal

Create the deterministic observation layer between M4.7 full-plant true state and all future control, protection and operator-display logic.

## Delivered

- canonical `InstrumentationSystemDefinition` and `InstrumentChannelDefinition`;
- finite `SignalRange` and deterministic `LinearSignalScale` primitives;
- separate `InstrumentationState` for lag/filter memory only;
- stable semantic `InstrumentSignalSourceCatalog` over `FullPlantSnapshot`;
- controller-facing `MeasuredSignalFrame` with no true-state reference;
- validity, quality, range and scaling diagnostics;
- exact deterministic first-order lag/filter evolution;
- explicit `Bias`, `Freeze`, `FailedLow`, `FailedHigh` and `Unavailable` sensor-fault seams;
- `InstrumentedFullPlantSolver` composing M4.7 physical evolution and M5.1 observation without duplicate physical integration;
- domain and simulation tests for topology, scaling, lag, quality, faults, determinism and true/measured-state separation;
- ADR 0039 and `docs/INSTRUMENTATION_SIGNAL_MODEL.md`.

## Explicitly deferred

- P/PI/PID controllers and actuator ownership;
- automatic synchronizer, AVR/excitation and turbine governor;
- reactor, steam, feedwater and condenser automatic loops;
- interlocks, trips, SCRAM and alarm logic;
- scenario-driven fault scheduling.

## Validation

Validated locally: build and complete automated test suite passed.
