# M4.7 — Full-Plant Steady-State Baseline

## Goal

Close the M4 reactor-to-grid phase with one deterministic full-plant state/snapshot boundary, configurable fixed-input reference operating point, long-run drift gate and gross plant-performance diagnostics before automatic control begins.

## Delivered

- `FullPlantState` immutable envelope over the existing thermofluid, rotor and electrical state owners;
- `FullPlantSolver` thin top-level delegate over M4.6 with no new physical integrator;
- `FullPlantSnapshot` as the canonical true-state boundary ready for M5 instrumentation;
- `FullPlantPerformanceDiagnostics` derived from the validated M4.6 heat-balance path;
- `FullPlantReferenceOperatingPoint` with fixed manual inputs, deterministic timestep and explicit acceptance criteria;
- `FullPlantLongRunRunner` / `FullPlantLongRunResult` with mass, coupled-energy, rotor-speed, electrical-output and closure-residual drift metrics;
- deterministic 1,000-step full-plant reference verification;
- explicit proof that criteria evaluation never corrects or trims physical state;
- ADR 0038 and `docs/FULL_PLANT_STEADY_STATE.md`.

## Explicitly deferred

- measured-signal/sensor layer;
- P/PI/PID controllers;
- automatic synchronizer, AVR/excitation and turbine governor;
- reactor/steam/feedwater automatic control loops;
- protection/interlock logic;
- operator control-room UI.

Those begin in M5.

## Validation

Validated by the user after successful local compilation; M4 gate complete.
