# M6.7 — Control-Room Integration & Performance Baseline

## Objective

Close the M6 gate by connecting the complete control-room presentation/command surface to the validated M5.7 runtime while proving rendering-cadence independence and bounded accelerated execution.

## Delivered

1. `IControlRoomRuntimeEngine` deterministic runtime boundary.
2. `IntegratedAutomaticOperationRuntimeEngine` adapter over the validated M5.7 solver/state/input chain.
3. `ControlRoomRuntimeCoordinator` implementing snapshot source + command dispatcher.
4. Run, pause and single-step runtime semantics.
5. Complete translation of current rod, MCP, turbine/generator, breaker, trip/protection and alarm command intents.
6. One-step pulse semantics for protection, breaker and annunciator commands.
7. Persistent immutable controller/setpoint input updates for normal operating commands.
8. Bounded accelerated batches with publication stride independent from simulation stepping.
9. Real M5.7 integration tests plus coordinator tests proving publication-cadence independence.
10. ADR 0052 and `docs/CONTROL_ROOM_INTEGRATION_PERFORMANCE.md`.

## Non-goals

- no invented default plant startup state;
- no scenario scheduler;
- no save/load format;
- no wall-clock simulation pacing;
- no physics in Avalonia.

## Validation status

**VALIDATED.** On 2026-07-21 the user explicitly confirmed that M6.7 compiles and the complete local test suite passes. M6.7 is the validated M6 baseline and the M6 gate is complete.
