## Delivered boundary

M10.5 formalizes two independent axes:

- training assistance: `Hidden` / `ChecklistOnly` / `Guided`, owned by the existing training/guidance layer and presentation-only;
- plant control authority: `Manual` / `Assisted` / `SupervisoryAutomatic`, owned by the M5 control domain.

The runtime publishes requested authority, effective authority, health (`Normal`, `Degraded`, `SuspendedByProtection`), optional degradation reason, semantic transition sequence and per-controller modes/setpoints through immutable Application presentation contracts.

`Manual` performs deterministic bumpless takeover by using each committed M5 controller's last output as its manual output. `Assisted` preserves the existing local controller modes/setpoints under operator authority. `SupervisoryAutomatic` is only effective when M10.6 has a valid bounded objective and required measured signals/protection state permit it.

Training assistance changes never change controller mode, physical state or scoring semantics.

## Replay semantics

Authority/objective intents are not forced into `ControlRoomCommandKind`. A dedicated `ScenarioAutomationIntentJournal` records accepted semantic intents. M9.1 full replay reapplies them at the same `N -> N+1` deterministic boundary and verifies the resulting frames fail-closed. The existing snapshot fingerprint schema is not changed.
