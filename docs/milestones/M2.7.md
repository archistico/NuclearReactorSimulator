# M2.7 — Void Feedback

## Goal

Connect the validated water/steam phase state to the compositional reactivity boundary without conflating vapor mass quality with vapor volumetric void fraction.

## Delivered

- strongly typed `VoidFraction` and signed `VoidFractionDifference`;
- strongly typed signed `VoidReactivityCoefficient`;
- `VoidReactivityFeedbackDefinition` with explicit reference void;
- deterministic `WaterSteamVoidFractionSolver`;
- homogeneous-equilibrium quality-to-volume conversion using M1.7 saturation densities;
- exact phase endpoints: subcooled liquid = 0 void, superheated vapor = 1 void;
- pure `VoidFeedbackSolver` producing named `Void` reactivity contributions;
- canonical multi-feedback composition through the validated M2.1 `ReactivityModel`;
- committed-state fixed-step coupling into M2.3 kinetics and M2.4 thermal power;
- unit, phase-mapping, feedback-sign, composition and pulse-segmentation determinism tests;
- ADR 0021 and `docs/VOID_FEEDBACK.md`.

## Explicitly deferred

- slip ratio and drift-flux void models;
- distributed/axial channel void fields;
- plant-specific RBMK void coefficient tables;
- pressure-drop/two-phase hydraulic closure beyond the M1 simplified model;
- same-step nonlinear multiphysics iteration.

## Validation

Validated via M2.7.1 — local build and full test suite passed after the test-only composition hotfix.
